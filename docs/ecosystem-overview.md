# Ecosystem Overview

## IronRidge Contractors

IronRidge Contractors is a fictional mid-sized commercial contractor and field operations company based in Indianapolis, Indiana. The business spans General Contracting, Field Services, Special Projects, and Operations Support, with realistic operating pressure around coordination, issue follow-up, and system maturity.

## The Four Demo Repositories

- `ops-visibility-demo` — executive reporting layer for project health, service backlog, closeout readiness, and risk
- `workflow-drag-reduction-demo` — internal support request workflow for intake, routing, approvals, and aging visibility
- `execution-infrastructure-demo` — lightweight execution tracker for commitments, blockers, ownership, and weekly review cadence
- `contractor-ops-system-demo` — field-aware operations tracker for issues, punch items, escalation, and closeout readiness

## How This Repository Fits

This repository is the field-aware operating layer. It shows how job-level issues and closeout dependencies could be tracked in a way that respects real contractor work instead of forcing the field into generic office-side systems.

## Shared Company Context

All four repos use the same IronRidge employee roster, project catalog, regions, divisions, and operating language so they read like connected parts of one internal systems environment.

## Portfolio Intent

This repo is intended to feel practical and believable: contractor-specific issue records, concise lifecycle documentation, and a screenshot-ready mock that reads like a useful internal tool.
